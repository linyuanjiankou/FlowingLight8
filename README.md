# STM32F103C8T6 多功能交互式信号演示系统

基于 STM32F103C8T6 (ARM Cortex-M3) 的嵌入式交互演示平台。本项目专注于 PWM 信号生成、ADC 模拟量采集与人机交互逻辑的实现，代码结构清晰，适合作为电子设计竞赛的基础模板或嵌入式开发的教学案例。

> **文档版本**: V1.0 
> **核心亮点**: 非阻塞按键状态机 + 模块化驱动 + 实时 OLED 反馈

---

## 📋 项目概览

本项目利用 STM32 的定时器 (TIM) 和 ADC 接口，构建了一个具有丰富视觉反馈的控制系统。系统支持 **4 种** 稳定的工作模式，通过 OLED 显示状态，并利用双按键进行参数调节。

### 核心特性
*   **精准 PWM 控制**：支持频率与亮度的动态调节。
*   **模拟量交互**：通过电位器实时控制输出信号。
*   **视觉反馈**：8路 LED 流水灯效果 + OLED 实时数据显示。
*   **低功耗设计**：非阻塞式按键扫描，避免主循环阻塞。

## 🛠️ 硬件环境

### 核心主控
*   **开发板**：STM32F103C8T6 (Blue Pill)
*   **主频**：72MHz
*   **资源**：64KB Flash / 20KB RAM

### 外设模块与接线
| 模块 | 型号/类型 | 连接引脚 (GPIO) | 备注 |
| :--- | :--- | :--- | :--- |
| **显示屏** | SSD1306 OLED | PB6 (SCL), PB7 (SDA) | I2C 通信, 地址 `0x78` |
| **按键 1** | KEY1 | PB3 | 频率/速度调节 |
| **按键 2** | KEY2 | PB13 | 亮度/方向调节 |
| **LED 阵列**| 8 x LED | PA0 - PA7 | 复用为 PWM 输出与指示灯 |
| **电位器** | Potentiometer| PB0 (ADC1_IN8) | 电压采集范围 0-3.3V |

## ⚙️ 功能模式详解

系统共有 **4 种** 工作模式，通过 **长按任意键** 进行循环切换。

### 1. 模式 0: PWM 频率调节 (`MODE_PWM_OUTPUT`)

- **功能描述**：通过定时器生成固定占空比（10%），频率可调的方波信号。
- **调节范围**：0.5Hz - 10Hz。
- **交互方式**：
    - 🔘 **短按 KEY1**：频率增加 0.5Hz (循环切换)。
    - 📺 **OLED 显示**：当前输出频率。

### 2. 模式 1: PWM 亮度调节 (`MODE_PWM_LIGHT`)

- **功能描述**：演示 PWM 调光原理，通过改变占空比控制 LED 视觉亮度（频率固定为高频）。
- **档位设置**：
    - 🌑 **暗**：10% 占空比
    - 🌓 **中**：50% 占空比
    - 🌞 **亮**：90% 占空比
- **交互方式**：
    - 🔘 **短按 KEY2**：切换亮度档位。

### 3. 模式 2: 电位器控制 (`MODE_PWM_ADC`)

- **功能描述**：读取外部模拟电压，实时映射为 PWM 输出频率。
- **算法亮点**：采用 **8点滑动平均滤波算法**，消除 ADC 采集噪声，输出平滑。
- **交互方式**：
    - 🎚️ **旋转电位器**：改变 PB0 引脚电压。
    - 📈 **OLED 显示**：实时显示当前电压值 (mV) 与映射后的频率 (Hz)。

### 4. 模式 3: 流水灯效果 (`MODE_FLOWINGLIGHT`)

- **功能描述**：控制 8 路 LED 实现动态流水效果。
- **交互方式**：
    - 🔘 **短按 KEY1**：切换流动速度 (快 200ms → 中 400ms → 慢 800ms)。
    - 🔘 **短按 KEY2**：切换流动方向 (正向 → 反向)。

### 🚀 快速开始

#### 1. 前置依赖

在开始之前，请确保你的电脑已安装以下软件：

- **Keil MDK-ARM (v5)**：必须安装，用于提供编译器和调试驱动。
- **Visual Studio Code**：用于编写代码。
- **STM32F1 Device Support Pack**：在 Keil 的 Pack Installer 中安装。

#### 2. 编译与烧录(二选一)

1. 克隆仓库：`git clone [你的仓库地址]`
2. 打开工程，检查 `main.c` 中的系统时钟是否配置为 **72MHz**。

**🅰️ 方案 A：使用 VSCode 进行开发与编译 (推荐)**

1. 使用 VSCode 打开项目根目录。
2. 确保已安装 **EIDE** 插件。
3. EIDE 会自动检测 Keil 的安装路径。如果报错，请在插件设置中手动指定 `armcc` 或 `armclang` 的路径。
4. 点击 EIDE 侧边栏的 **“构建 (Build)”** 按钮进行编译。
5. 点击 **“烧录 (Download)”** 将程序下载至开发板。

**🅱️ 方案 B：使用 Keil 进行调试**

1. 双击打开 `FlowingLight8.uvprojx` 工程文件。
2. 点击 **Rebuild** 确保工程配置无误。
3. 连接 ST-Link 调试器，点击 **Start/Stop Debug Session** 进入调试模式。
    - _注意：请确保 Keil 中的 Debug 选项已配置为 ST-Link Debugger。_

### 4. 硬件连接

- 连接 OLED 屏幕至 PB6/PB7。
- 连接 8 个 LED 至 PA0-PA7（注意共阳/共阴接法，代码默认低电平点亮）。
- 将电位器中间引脚连接至 PB0，两端分别接 3.3V 和 GND。

## 📊 关键技术实现

- **非阻塞按键状态机**：  
    采用 `KEY_Free` -> `KEY_Pressed` -> `KEY_Released` 的状态流转，精准识别短按与长按事件，不占用 CPU 资源进行死等。
- **滑动平均滤波 (ADC)**：  
    定义长度为 8 的环形缓冲区，每次读取新值覆盖旧值并计算平均值，有效抑制了电位器的接触噪声。
- **位带操作 (Bit-Banding)**：  
    利用 STM32 位带特性直接操作 GPIO 寄存器，实现 LED 点亮的原子操作，提高流水灯运行效率。
